## PESQUISA

### O que é o React?

React é uma **biblioteca JavaScript** criada pelo **Facebook (Meta)** em **2011**, focada na **construção de interfaces de usuário (UI)**, principalmente **Single Page Applications (SPA)**.

### Por que o React foi criado?

O React foi criado com o objetivo de **otimizar a atualização e a sincronização de atividades simultâneas**, como o feed de notícias da rede social, além de **melhorar a manutenção de código** por meio da criação de *views* declarativas e componentes reutilizáveis.

### Por que utilizar o React?

O React é amplamente utilizado porque facilita a criação de interfaces modernas, rápidas e organizadas, mesmo em aplicações grandes e complexas. Um dos principais motivos para sua adoção é a forma como ele organiza o código.

#### Componentização

O React divide a interface em componentes reutilizáveis, como botões, menus e formulários. Isso deixa o código mais organizado, facilita mudanças e evita retrabalho, já que um mesmo componente pode ser usado em várias partes da aplicação.

#### Virtual DOM e desempenho

O React usa o Virtual DOM, que atualiza apenas o que realmente mudou na tela. Isso torna a aplicação mais rápida e fluida, principalmente em sistemas com muitas interações do usuário.

#### Facilidade para criar interfaces dinâmicas

Com o controle de estado, o React atualiza a tela automaticamente quando os dados mudam. Isso facilita a criação de páginas interativas, sem precisar recarregar o site.

#### Popularidade e mercado de trabalho

O React é muito utilizado por grandes empresas e muito requisitado no mercado. Isso garante bastante conteúdo de estudo, ajuda da comunidade e boas oportunidades profissionais.

#### Flexibilidade e comunidade

O React é flexível e pode ser usado em projetos de diferentes tamanhos.  
Além disso, possui uma comunidade ativa e atualizações constantes, o que garante evolução e suporte contínuo.

#### Comparativo de Aplicações

- **Angular:** O React oferece uma abordagem mais simples e flexível, evitando a rigidez e a curva de aprendizado elevada do Angular.
- **Vue.js:** O React se destaca por sua maior adoção, comunidade mais ativa e mais oportunidades profissionais.
- **Svelte:** O React possui maior maturidade e um ecossistema mais completo, sendo uma escolha mais estável e segura para projetos reais.

---

### Em quais cenários ele não é indicado?

O React é muito poderoso, mas não é a melhor escolha para todo tipo de projeto. Em sites muito simples (como landing pages básicas) não é recomendado o uso, pois o site é só uma página institucional com pouco conteúdo dinâmico e sem interações complexas. O uso do React pode ser **overengineering** (engenharia em excesso).

O React também não é tão recomendado para sistemas muito pequenos ou scripts rápidos, como:

- Um painel simples
- Um formulário com validação básica
- Uma automação web pequena

Nesses casos, o React pode ser pesado demais. Às vezes, **JavaScript puro** ou um framework mais leve, como o **Vue.js**, resolve melhor.

> O React é perfeito para aplicações grandes e interativas.
